# 탐지 모델

## 기본 원칙

봇 여부만 판단해서는 부족합니다. 정상 검색봇도 서버를 과도하게 사용할 수 있고, 실제 사용자의 갑작스러운 유입도 봇처럼 보일 수 있습니다. 따라서 서로 다른 세 값을 계산합니다.

| 점수 | 의미 |
|---|---|
| 신뢰 점수 | 검증된 검색엔진, 정상 세션, 관리자 등 신원을 얼마나 신뢰할 수 있는가 |
| 봇 가능성 | 자동화된 클라이언트처럼 행동하는 정도 |
| 자원 비용 | 요청이 PHP, DB, 메모리와 네트워크에 만드는 실제 부담 |

## 신원 신호

- 공식 IP 대역과 정방향·역방향 DNS로 검증된 검색봇
- User-Agent와 IP·ASN 조합
- 데이터센터·클라우드 사업자 IP 여부
- 정상적으로 이어지는 서명 쿠키와 세션
- 관리자가 등록한 허용·차단 규칙

User-Agent는 쉽게 위조되므로 단독 판정 근거로 사용하지 않습니다.

## 행동 신호

- 사람이 보기 어려운 일정한 요청 간격
- CSS, JavaScript, 이미지 없이 HTML만 순회
- 쿠키와 세션을 유지하지 않음
- 짧은 시간에 많은 고유 URL 접근
- 검색어, 페이지 번호, 게시판과 회원 ID를 순차 증가
- `robots.txt` 제한 경로 반복 접근
- 404, 캐시 미스와 연결 중단 비율 증가
- 브라우저 헤더 조합의 불일치
- 체류시간 없이 다음 페이지를 즉시 요청

## 자원 비용 신호

### 요청 단위

- 경로별 기본 비용 등급
- 요청 처리시간과 upstream 응답시간
- 전송한 응답 크기
- 캐시 적중·미스 여부
- 업로드 크기와 실행시간

### 서버 단위

- PHP-FPM active process와 `max_children` 접근률
- MySQL 연결 수, 느린 쿼리와 잠금 대기
- Redis hit/miss와 메모리 사용량
- CPU, load average, 메모리, swap과 디스크 대기
- 5xx, timeout과 연결 거부 증가율

## 초기 비용표 예시

| 요청 | 기본 비용 |
|---|---:|
| 정적 파일 | 1 |
| 캐시된 게시물 | 2 |
| 게시판 목록·상세 | 4 |
| 회원 프로필 | 6 |
| 깊은 페이지네이션 | 8 |
| 검색 | 10 |
| 로그인·비밀번호 요청 | 12 |
| 업로드·이미지 처리 | 15 |

실제 upstream 지연, DB 포화와 cache miss가 발생하면 동적으로 가중치를 더합니다. 초기에는 설명 가능한 규칙 기반 점수를 사용하고 충분한 사건 데이터가 쌓이기 전에는 머신러닝을 도입하지 않습니다.

## 판정표

| 신뢰 | 봇 가능성 | 자원 비용 | 조치 |
|---|---|---|---|
| 높음 | 낮음 | 낮음 | 허용 |
| 높음 | 무관 | 높음 | 속도 제한과 Retry-After |
| 낮음 | 중간 | 낮음 | 관찰과 단기 세션 발급 |
| 낮음 | 중간 | 높음 | 브라우저 검증 |
| 낮음 | 높음 | 높음 | 임시 차단 |
| 분산됨 | 높음 | 서버 포화 | Cloudflare 비상 전환 |

## 오탐 방지

- 신규 설치 후 최소 관찰 기간을 두고 자동 차단을 비활성화합니다.
- 한 가지 신호만으로 영구 차단하지 않습니다.
- IP 하나에 여러 사용자가 존재할 수 있으므로 세션·ASN·경로 행동을 함께 봅니다.
- 검증된 검색봇도 무제한 허용하지 않고 친화적인 속도 제한을 적용합니다.
- 차단에는 만료시간을 두고 자동 재평가합니다.
- 모든 결정에 사람이 읽을 수 있는 근거를 저장합니다.
- 자동 대응 전후의 사용자 오류율과 이탈 신호를 비교합니다.

## GnuBoard·WordPress 초기 프로필

첫 버전은 범용 AI 모델보다 애플리케이션별 고비용 경로 프로필로 정확도를 확보합니다.

- 검색
- 로그인·회원가입·비밀번호 재설정
- 게시판 목록과 깊은 페이지
- 회원·작성자 프로필
- 댓글·게시물 작성
- 파일 업로드와 이미지 변환
- XML-RPC, REST API와 피드
- 관리자 경로

정확한 경로와 비용은 실제 GnuBoard·WordPress 설치본의 부하 테스트로 확정합니다.

## 사건 기록 예시

```text
10:03:10 검색 경로 요청 비용이 평시 대비 6.2배 증가
10:03:12 14개 IP가 세션 없이 2,800개 검색어 순회
10:03:14 PHP-FPM active 88%, MySQL p95 740ms
10:03:15 검색 경로 10 req/min 제한 적용
10:03:22 PHP-FPM active 54%, MySQL p95 110ms
판정: 분산 검색 스크래퍼, Cloudflare 전환 불필요
```

이 설명 가능성이 제품 신뢰의 핵심입니다.
