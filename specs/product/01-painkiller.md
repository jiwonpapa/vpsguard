# 페인킬러

## 해결할 강제 선택

| 현재 선택 | 얻는 것 | 잃는 것 |
|---|---|---|
| VPS 원본 직접 연결 | 국내 사용자의 빠른 응답 | 봇, 스크래핑, 자원 고갈과 원본 노출 |
| Cloudflare 상시 프록시 | 엣지 방어와 원본 은닉 | 국내 체감 속도와 원본 직접 연결의 단순성 |
| 고사양 VPS로 증설 | 일시적인 처리 여유 | 매달 반복되는 비용, 근본 원인 미해결 |
| 운영자 수동 대응 | 세밀한 판단 | 야간 대응, 로그 분석, 차단과 복구 실수 |

VPS Guard의 페인킬러는 **보안 제품을 하나 더 설치하는 것**이 아닙니다. 정상 사용자에게는 직접 연결 성능을 유지하면서, 사이트를 죽이는 자동화 트래픽만 먼저 줄이고, 로컬 방어 한계를 넘을 때만 외부 방어망을 사용하는 것입니다.

## 주요 사용자

- 1~4GB VPS 한 대에서 커뮤니티나 콘텐츠 사이트를 운영하는 개인·소규모 사업자
- GnuBoard, WordPress, PHP-FPM과 MySQL을 직접 운영하는 사용자
- 전담 서버 관리자가 없고 장애 대응 시간을 줄이고 싶은 개발자
- Cloudflare 상시 프록시의 국내 체감 속도가 불만인 운영자
- 봇 때문에 VPS를 증설했지만 실제 사용자 수는 많지 않은 사이트

## 실제 비용 누수

### 1. 불필요한 VPS 증설

AI 크롤러와 검색 스크래퍼가 검색, 깊은 페이지 이동, 회원 조회처럼 DB 비용이 큰 경로를 반복하면 실제 사용자는 적어도 PHP-FPM과 DB가 포화됩니다. 운영자는 원인을 찾기 전에 상위 사양으로 증설하기 쉽습니다.

### 2. 장애와 이탈

사이트가 느려지거나 연결이 끊기면 광고·구독·회원 활동이 줄고 검색엔진 평가에도 악영향을 줄 수 있습니다. 작은 사이트일수록 장애 원인 분석과 복구에 운영자 시간이 직접 소모됩니다.

### 3. 상시 방어의 성능 비용

공격이 없는 대부분의 시간에도 외부 프록시를 계속 통과하면 정상 사용자가 매 요청마다 성능 비용을 지불합니다. 국내 최적화형 기업 서비스는 일반 개인·소형 사이트가 감당하기 어렵다는 것이 이 제품의 출발점입니다.

### 4. 수동 운영 비용

접근 로그 확인, IP 차단, Cloudflare 프록시 전환, 방화벽 수정과 복구를 사람이 수행하면 늦고 실수하기 쉽습니다. 특히 원본 방화벽을 잘못 잠그면 정상 사용자나 SSH 관리 접속까지 차단할 수 있습니다.

## 제품이 절감하는 비용

```text
월간 절감 가치
= 회피한 VPS 증설 비용
+ 회피한 유료 보안 서비스 비용
+ 줄어든 장애 시간의 가치
+ 줄어든 운영자 대응 시간
+ 성능 저하로 잃지 않은 정상 사용자 가치
```

절감액을 과장하지 않기 위해 대시보드에는 다음 근거를 함께 표시합니다.

- 차단·지연한 고비용 요청 수
- 보호 전후 PHP-FPM 사용률과 DB 지연
- 직접 연결과 프록시 연결의 응답시간 비교
- 자동 대응으로 줄어든 장애 지속시간
- 상위 VPS로 증설하지 않고 확보한 자원 여유

## 정확한 제품 약속

> VPS Guard는 모든 봇을 없애는 제품이 아니라, **정상 사용자의 속도를 유지하면서 자동화 트래픽이 만드는 불필요한 서버 비용과 장애를 줄이는 제품**입니다.

## 하지 않을 약속

- 서버 회선을 이미 포화시킨 대규모 네트워크 공격을 로컬 프로그램만으로 막는다고 약속하지 않습니다.
- 봇과 사람을 100% 구분한다고 약속하지 않습니다.
- 애플리케이션 자체의 느린 쿼리와 잘못된 코드를 자동으로 수정하지 않습니다.
- Cloudflare나 VPS 사업자의 API 장애까지 통제할 수 있다고 약속하지 않습니다.
